---

## 📢 1. 사용자 요구사항 분석

**User Needs**:

- 글 작성 중 **문맥에 맞는 객관적 정보(논문, 통계)**가 필요함.
- 단순 생성이 아닌 **"검증된 데이터"**와 **"출처(Source)"**가 필수.
- **Web Search LLM API**를 활용하여 완성도를 높이고 싶음.

**Core Value**: **"Trustworthiness" (신뢰성)** & **"Groundedness" (근거 기반)**

---

## 💡 2. 아이디어 제안 및 기술 검토

---

### 👶 **Jun** (주니어 개발자)

> "요즘 Perplexity 많이 쓰잖아요? 그걸 붙이면 어떨까요?"

**💡 아이디어: "Perplexity Integration"**

- **방식**: `Perplexity API (sonar-reasoning)`를 사용하여 질문하면 출처가 달린 답변을 받아옴.
- **장점**: 구현이 매우 쉽고, 인용(Citation) 품질이 이미 검증됨.
- **단점**: 커스터마이징이 어렵고, 특정 도메인(예: 논문만 검색) 필터링이 제한적일 수 있음.

---

### 👨‍💻 **Alex** (백엔드 시니어)

> "RAG에 최적화된 검색 API를 직접 컨트롤하는 게 낫습니다."

**💡 아이디어: "Scholar Agent" (with Tavily/Serper)**

- **방식**: **Tavily Search API** (LLM용 검색 엔진) 사용.
- **로직**:
  1. 사용자 문맥 파악 -> 검색 쿼리 생성 ("2024 AI market size statistics")
  2. **Domain Filter**: `scholar.google.com`, `*.edu`, `*.gov`, `arxiv.org` 등으로 검색 대상 한정.
  3. LLM이 검색 결과(Raw Content)를 읽고 요약하여 에디터에 삽입.
- **장점**: **정보의 출처를 우리가 완벽하게 통제**할 수 있음. (예: "블로그 제외하고 논문만 찾아")

---

### 👩‍💻 **Mia** (프론트엔드 시니어)

> "글쓰기 흐름을 끊지 않는 게 중요해요. 텍스트 드래그로 바로 찾게 하죠."

**💡 아이디어: "Contextual Fact-Check" (문맥 팩트체크)**

- **UX**: 사용자가 쓴 문장을 드래그 -> "🔍 근거 찾기" 버튼 클릭 -> 사이드바에 관련 논문/통계 리스트업 -> 클릭 시 내용을 인용구로 삽입.
- **장점**: 사용자가 능동적으로 "이 부분에 근거가 필요해"라고 느낄 때 즉시 지원.

---

### 🎨 **Sophie** (UX 전문)

> "정보의 신뢰도를 시각적으로 보여줘야 합니다."

**💡 아이디어: "Trust Badge & Footnote System"**

- **UX**: 외부 정보 삽입 시 자동으로 **각주([1])**가 생성되고, 해당 출처의 신뢰도(예: "Science지 인용됨", "공식 정부 기관")를 뱃지로 표시.
- **장점**: "이 정보는 믿을 수 있다"는 확신을 UX로 전달.

## 🗳️ 4. 최종 솔루션 도출: "Deep Scholar" (딥 스칼라)

참석자 전원 만장일치로 **Alex(검색 제어)** + **Mia(드래그 UX)** + **Jun(Perplexity급 성능)**을 결합한 통합 모델 채택.

### 🏆 결정된 기능: **"Deep Scholar" (심층 연구 어시스턴트)**

1.  **기술 스택**: **Tavily API** (학술/뉴스 필터링 특화) + **Gemini 3.0 Flash** (팩트 추출 및 요약)
2.  **작동 시나리오**:
    - 사용자가 `/research` 명령어를 입력하거나 텍스트 드래그.
    - "이 주장에 대한 최신 통계 찾아줘" 라고 요청.
    - **Agent**가 `*.edu`, `arxiv.org` 등 신뢰할 수 있는 도메인에서 실시간 검색.
    - 결과를 **[출처] + [핵심 요약]** 카드로 보여주고, 클릭 시 에디터에 **각주 달린 텍스트**로 삽입.

**기술 리더 총평**:
"기존 Shadow Writer가 **'내부 지식(Internal RAG)'**을 활용한 속도전이라면, Deep Scholar는 **'외부 지식(External Search)'**을 활용한 신뢰전입니다. 이 두 날개가 합쳐지면 PRISM Writer는 완벽한 저작 도구가 될 것입니다."
