> 코드 수정 범위도 작습니다.

**👨‍💻 주니어 개발자 의견:**

> "참고자료" 탭 상단에 "📤 자료 업로드" 버튼을 추가하면 어떨까요?
> 클릭 시 모달창으로 드래그&드롭 영역이 나타나는 방식이요.

**🎨 UX/UI 전문가 의견:**

> 두 가지 방안을 제안드립니다:
>
> | 방안                    | 장점                    | 단점           |
> | ----------------------- | ----------------------- | -------------- |
> | **A. 탭 상단 버튼**     | 눈에 잘 띔, 접근성 좋음 | 화면 공간 차지 |
> | **B. 검색창 옆 아이콘** | 미니멀, 공간 효율적     | 발견성 낮음    |
>
> **권장안**: 방안 A (초보 사용자도 쉽게 찾을 수 있음)

**✅ 합의사항:**

- "참고자료" 탭 상단에 "📤 자료 업로드" 버튼 배치
- 클릭 시 모달 또는 확장 영역으로 드래그&드롭 UI 표시

### 안건 3: 업로드 파일 목록 및 내용 표시

**🎨 UX/UI 전문가 의견:**

> 사용자가 "어떤 자료를 올렸는지"를 빠르게 파악해야 합니다.
>
> **권장 UI 구성:**
>
> ```
> ┌─────────────────────────────────────┐
> │ 📤 자료 업로드                [+ 추가]│
> ├─────────────────────────────────────┤
> │ 📄 보고서_최종.pdf     ✅ 준비완료   │
> │ 📄 참고문헌.docx       ⏳ 처리중     │
> │ 📄 메모.txt            ❌ 오류       │
> └─────────────────────────────────────┘
> ```
>
> 각 항목에 표시할 정보:
>
> - 파일명 (잘림 처리)
> - 상태 배지 (준비완료/처리중/오류)
> - 삭제 버튼 (hover 시 표시)

**🧑‍💻 시니어 개발자 의견:**

> 이미 `DocumentList` 컴포넌트가 있습니다.
> 상태 값은 DB의 `status` 필드에서 가져옵니다:
>
> - `pending` → ⏳ 대기중
> - `processing` → ⏳ 처리중
> - `ready` → ✅ 준비완료
> - `error` → ❌ 오류

**✅ 합의사항:**

- 파일명 + 상태 배지로 심플하게 표시
- 상세 내용(청크 수, 토큰 사용량 등)은 숨김 (필요 시 클릭으로 확장)

---

### 안건 4: 파일 분류/저장 방식 노출 여부

**🎨 UX/UI 전문가 의견:**

> **결론: 사용자에게 보여줄 필요 없음**
>
> 이유:
>
> 1. 기술적 세부사항(임베딩, 벡터, 청킹)은 사용자 혼란 유발
> 2. 사용자는 "파일 올림 → 검색됨"만 알면 충분
> 3. 복잡한 UI는 이탈률 증가
>
> **숨길 정보:**
>
> - 768차원 벡터
> - 청크 개수
> - 토큰 사용량
> - Storage 경로

**🧑‍💻 시니어 개발자 의견:**

> 동의합니다. 다만 "처리 중" 상태일 때 프로그레스 표시는 필요합니다.
> "분석 중..." 정도의 메시지만 보여주면 됩니다.

**👨‍💻 주니어 개발자 의견:**

> 관리자용으로는 상세 정보가 필요할 수 있어요.
> 일반 사용자에게는 숨기고, 개발자 콘솔이나 관리 페이지에서만 보이게 하면 어떨까요?

**✅ 합의사항:**

- 일반 사용자: 파일명 + 상태만 표시
- 기술적 세부사항은 완전히 숨김
- 관리자용 상세 정보는 별도 고려 (추후)
