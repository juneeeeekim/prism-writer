## 1. 조사 결과 요약

> ✅ **결론: 대기업들도 동일하게 토큰/메시지/단어 기반으로 사용량을 제한하고 있습니다.**

모든 주요 LLM 서비스 제공 업체는 **사용량 제한 기반 요금제**를 운영하고 있으며, 제한 방식은 크게 3가지로 분류됩니다:

| 제한 방식     | 사용 기업                  | 특징                         |
| ------------- | -------------------------- | ---------------------------- |
| **메시지 수** | OpenAI, Anthropic          | 시간당/일당 메시지 횟수 제한 |
| **토큰 수**   | OpenAI API, Google API     | 월간 토큰 할당량             |
| **단어 수**   | Jasper, Copy.ai, Grammarly | 월간 단어 생성 한도          |

## 2. 대기업별 상세 분석

### 2.1 OpenAI (ChatGPT)

| 플랜     | 가격    | 제한 방식   | 상세 제한                                       |
| -------- | ------- | ----------- | ----------------------------------------------- |
| **Free** | $0      | 메시지/시간 | GPT-4o: ~16메시지/3시간                         |
| **Plus** | $20/월  | 메시지/시간 | GPT-4o: ~80메시지/3시간, GPT-4: ~40메시지/3시간 |
| **Pro**  | $200/월 | 무제한      | 무제한 + 우선 처리                              |

**PM 의견**:

> "OpenAI는 토큰이 아닌 **메시지 수로 제한**합니다. 사용자가 이해하기 쉬운 방식이지만, 긴 대화는 토큰을 더 소모하므로 실제 비용과 괴리가 있을 수 있습니다."

**시니어 개발자 의견**:

> "**시간 기반 리셋**(3시간마다)은 흥미로운 전략입니다. 사용자가 몰아서 사용하는 것을 방지하면서도 일정 시간 후 다시 사용할 수 있도록 합니다."

---

### 2.2 Anthropic (Claude)

| 플랜     | 가격   | 제한 방식    | 상세 제한                        |
| -------- | ------ | ------------ | -------------------------------- |
| **Free** | $0     | 메시지/5시간 | ~10-15메시지/5시간 (추정)        |
| **Pro**  | $20/월 | 메시지/5시간 | ~45메시지/5시간 (짧은 대화 기준) |

**주요 특징**:

- **5시간 리셋 주기** (OpenAI보다 김)
- **대화 길이에 따라 가변**: 긴 대화는 더 적은 메시지 허용
- 200,000 토큰 컨텍스트 윈도우

**AI/ML 엔지니어 의견**:

> "Claude는 **대화 길이를 고려한 가변 제한**을 적용합니다. 이는 기술적으로 더 공정하지만, 사용자 입장에서는 예측하기 어려울 수 있습니다."

---

### 2.3 Google (Gemini)

| 플랜         | 가격      | 제한 방식   | 상세 제한                        |
| ------------ | --------- | ----------- | -------------------------------- |
| **Free**     | $0        | 프롬프트/일 | ~30 프롬프트/일                  |
| **Advanced** | $19.99/월 | 프롬프트/일 | 100 프롬프트/일, 1,000 이미지/일 |

**주요 특징**:

- **일일 리셋**(OpenAI/Claude와 다름)
- 1,000,000 토큰 컨텍스트 윈도우 (가장 큼)
- Google One 스토리지 2TB 등 번들 제공

**PM 의견**:

> "Google은 **일일 프롬프트 수**로 제한합니다. 단순하고 이해하기 쉬운 모델입니다. 사용자가 '오늘 몇 번 사용했는지' 직관적으로 알 수 있습니다."

### 2.4 Microsoft (Copilot)

| 플랜             | 가격   | 제한 방식 | 상세 제한                  |
| ---------------- | ------ | --------- | -------------------------- |
| **Free**         | $0     | 제한적    | 기본 기능만                |
| **Pro**          | $20/월 | 요청/분   | ~18 요청/분, 100 이미지/일 |
| **M365 Copilot** | $30/월 | 요청/분   | Excel 함수: 60회/5분       |

**주요 특징**:

- **분당 요청 수 제한** (Rate Limiting)
- 기업 고객 대상 별도 협상

**DevOps 엔지니어 의견**:

> "Microsoft는 **분당 제한(Rate Limiting)**을 강조합니다. 이는 서버 과부하 방지에 효과적이지만, 월간 총량 제한과 병행해야 비용 통제가 가능합니다."

## 3. 제한 방식별 비교 분석

### 3.1 제한 방식 장단점

| 방식               | 장점               | 단점                    | 사용 기업         |
| ------------------ | ------------------ | ----------------------- | ----------------- |
| **메시지/시간**    | 직관적, 예측 가능  | 긴 대화에 불리          | OpenAI, Anthropic |
| **프롬프트/일**    | 가장 단순          | 정교한 비용 통제 어려움 | Google            |
| **토큰/월**        | 정확한 비용 추적   | 사용자 이해 어려움      | API 과금          |
| **단어/월**        | 콘텐츠 생성에 적합 | 대화형에 부적합         | Jasper, Copy.ai   |
| **AI 프롬프트/월** | 범용적             | 응답 길이 고려 안됨     | Grammarly         |

### 3.2 리셋 주기 비교

| 서비스    | 리셋 주기 | 전략적 의미            |
| --------- | --------- | ---------------------- |
| OpenAI    | 3시간     | 단기 몰아쓰기 방지     |
| Anthropic | 5시간     | 더 여유로운 사용       |
| Google    | 24시간    | 일일 예산 개념         |
| Grammarly | 월간      | 장기 계획 가능         |
| Notion    | 리셋 없음 | 시험 후 유료 전환 유도 |

## 4. PRISM Writer 적용 권장 사항

### 4.1 전문가 투표 결과

| 제한 방식 | 투표     | 이유                                    |
| --------- | -------- | --------------------------------------- |
| 토큰/월   | ⭐⭐⭐⭐ | 정확한 비용 추적, 기술적 일관성         |
| 메시지/일 | ⭐⭐⭐   | 사용자 친화적, 단순함                   |
| 단어/월   | ⭐⭐     | 글쓰기 도구에 적합하나 비용 예측 어려움 |

### 4.2 권장 모델: **하이브리드 (토큰 + 일일 요청)**

**PM 제안**:

> "내부적으로는 **토큰 기반**으로 비용을 추적하되, 사용자에게는 **일일 AI 요청 횟수 + 월간 총량**으로 표시하는 것을 권장합니다. 이는 Google과 Grammarly의 장점을 결합한 방식입니다."

```
사용자 화면: "오늘 5/50회 사용 | 이번 달 1,234/10,000 토큰"
내부 추적: input_tokens + output_tokens → 비용 계산
```

### 4.3 등급별 권장 할당량 (수정안)

| 등급        | 일일 요청 | 월간 토큰 | 참고 서비스         |
| ----------- | --------- | --------- | ------------------- |
| **pending** | 0         | 0         | -                   |
| **free**    | 20회      | 10,000    | Grammarly Free 참고 |
| **premium** | 100회     | 50,000    | ChatGPT Plus 참고   |
| **special** | 무제한    | 200,000   | 헤비 유저           |
| **admin**   | 무제한    | 무제한    | -                   |

**AI/ML 엔지니어 의견**:

> "무료 등급의 **일일 20회**는 Grammarly(100회/월 ÷ 5일 = 20회/일)와 유사한 수준입니다. 서비스 체험은 가능하되 무분별한 사용은 방지됩니다."

## 5. 추가 비용 최적화 인사이트

### 5.1 대기업들의 비용 절감 전략

| 전략               | 사용 기업          | 효과                      |
| ------------------ | ------------------ | ------------------------- |
| **응답 캐싱**      | 모든 기업 (추정)   | 30-50% 비용 절감          |
| **모델 티어링**    | OpenAI, Anthropic  | 간단한 작업은 저렴한 모델 |
| **컨텍스트 압축**  | Claude, GPT        | 긴 대화 토큰 절약         |
| **출력 토큰 제한** | OpenAI (4,096 max) | 과도한 응답 방지          |

### 5.2 Fair Use 정책 (참고)

Notion AI와 같은 "무제한" 플랜도 실제로는 **Fair Use 정책**을 적용합니다:

- 비정상적 사용 패턴 탐지
- 자동화/봇 사용 금지
- 상업적 대량 생성 제한

**보안 전문가 의견**:

> "무제한 플랜이라도 **악용 방지 로직**은 필수입니다. Notion처럼 'Fair Use' 조항을 마련하고, 이상 사용 탐지 시스템을 구축해야 합니다."

## 6. 회의 결론

### 6.1 핵심 인사이트

1. ✅ **모든 대기업이 사용량 제한을 적용** - 우리만의 방식이 아님
2. ✅ **토큰 기반이 가장 정확** - 비용 추적에 필수
3. ✅ **사용자 친화적 표시 필요** - 토큰 대신 횟수/단어로 표시
4. ✅ **리셋 주기는 전략적 선택** - 일일 or 월간

### 6.2 PRISM Writer 최종 권장안

```
┌─────────────────────────────────────────────────────┐
│  내부: 토큰 기반 추적 (정확한 비용 계산)              │
│  ↓                                                  │
│  사용자 표시: "오늘 N회" + "이번 달 N토큰"           │
│  ↓                                                  │
│  리셋: 일일 요청 = 매일 자정 / 월간 토큰 = 매월 1일  │
└─────────────────────────────────────────────────────┘
```
