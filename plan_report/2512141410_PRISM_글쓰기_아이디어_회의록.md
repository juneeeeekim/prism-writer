**주제:** PRISM 엔진을 활용한 **"RAG 기반 글쓰기 도구"** 로의 피벗(Pivot) 아이디어 회의

---

## 0. Context Analysis (디렉터 의도 분석)

기존 분석했던 문서들(`그록`, `클로드`, `GPT`, `초안`, `확정안`)의 기술적 기반(Clean Arch, Hexagonal, Vector DB)은 그대로 유지하되, **최종 결과물(Output)** 의 형태가 변경되었습니다.

- **AS-IS (기존):** 질문을 던지면 답을 주는 **"챗봇 (Chatbot)"**
- **TO-BE (변경):** 자료를 분석하여 글의 구조와 내용을 잡아주는 **"저작 도구 (Writing Assistant)"**

> **핵심 목표:** "글을 쓸 때의 **구조(Structure)** 와 **내용(Content)** 의 초석을 RAG가 제공해야 한다."

---

## 1. How Might We (우리가 어떻게 해결할 수 있을까?)

팀원들은 이 목표를 달성하기 위해 다음과 같은 핵심 질문(HMW)을 도출했습니다.

1.  **HMW 1 (구조):** 어떻게 하면 RAG가 수많은 문서 파편들을 조합하여 **"논리적인 글의 목차(Outline)"** 를 자동으로 제안할 수 있을까?
2.  **HMW 2 (내용):** 사용자가 특정 문단을 작성할 때, 어떻게 하면 **"딱 맞는 참고자료"** 를 옆에서 바로 보여줄 수 있을까?
3.  **HMW 3 (UX):** 어떻게 하면 "검색"과 "글쓰기"가 분리되지 않고 **"하나의 흐름"** 으로 느껴지게 할 수 있을까?

---

## 2. 아이디어 브레인스토밍 (Idea Generation)

각 역할별로 자유롭게 아이디어를 제시했습니다.

### 🏗️ System Architect

- **"Draft-Reference Link 모델":** 단순히 글만 저장하는 게 아니라, 글의 각 문단이 **어떤 문서의 어떤 청크(Chunk)를 참조했는지** 메타데이터로 연결해야 합니다. 나중에 "출처 확인" 버튼만 누르면 원문이 떠야 합니다.

### 🤖 AI/ML Engineer

- **"Structural RAG (구조적 RAG)":**
  - 사용자가 주제를 입력하면, 관련 문서들의 **헤더(H1, H2)** 만 먼저 검색해서 수집합니다.
  - LLM에게 "이 헤더들을 보고 가장 논리적인 **목차(Table of Contents)** 를 생성해줘"라고 요청하는 2단계 파이프라인이 필요합니다.
- **"Context-Aware Autocomplete":** 단순 단어 완성이 아니라, "이 다음 내용은 참고문서 A의 내용을 요약해서 써줘"라고 하면 써주는 기능.

### 🎨 Frontend / UX Designer

- **"Dual Pane Editor (듀얼 페인 에디터)":**
  - **왼쪽:** 마크다운 에디터 (글쓰기 공간)
  - **오른쪽:** RAG 어시스턴트 (자료 추천, 검색, 구조 제안)
  - 채팅창이 메인이 아니라, **에디터가 메인**이어야 합니다.
- **"Block-based Reference":** 노션(Notion)처럼 블록 단위로 글을 쓰되, `/rag` 명령어를 치면 해당 블록에 참고할 만한 내 문서가 팝업으로 떠야 합니다.

### ⚙️ Backend Engineer

- **"Snapshot Versioning":** 글을 쓰는 도중 RAG가 추천해준 내용이 마음에 안 들 수 있습니다. "RAG 적용 전/후"를 쉽게 되돌릴 수 있는 스냅샷 기능이 필수입니다.

## 3. 상호 검토 및 보완 (Critique)

- **비판 (Frontend):** "채팅 UI에서 글을 써주는 것"과 "에디터에서 글을 쓰는 것"은 완전히 다릅니다. 기존 로드맵의 `ChatInterface` 컴포넌트는 폐기하고 `EditorInterface`를 새로 설계해야 합니다.
- **보완 (Security):** 글쓰기 도구는 "저장"이 매우 빈번합니다. RAG 검색 쿼리가 너무 많이 발생하면 비용이 폭주할 수 있습니다. **"명시적 요청(버튼 클릭) 시에만 RAG 가동"** 으로 제한해야 합니다.

## 5. 최종 방향성 정리 (Final Direction)

**"PRISM Writer: 내 지식 기반의 지능형 저작 도구"**

우리는 PRISM을 단순 챗봇이 아닌, **"자료 조사(Research)와 집필(Writing)이 통합된 워크스페이스"** 로 정의합니다.

1.  **구조 잡기 (Structure):** 사용자가 주제를 던지면, RAG가 내 문서들을 훑어보고 **"목차(Outline)"** 를 제안합니다.
2.  **내용 채우기 (Content):** 목차의 각 챕터를 클릭하면, 해당 챕터에 필요한 **"핵심 증거(Evidence)"** 들을 우측 패널에 띄워줍니다.
3.  **글쓰기 (Drafting):** 사용자는 우측의 증거를 드래그&드롭하거나 "이걸로 써줘"라고 요청하여 왼쪽 에디터에 글을 채웁니다.
