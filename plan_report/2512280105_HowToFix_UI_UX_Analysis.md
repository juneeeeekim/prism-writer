# How to Fix - UI/UX 검토 및 개선 제안 보고서

**작성일**: 2025-12-28  
**작성자**: Tech Lead & Design Team  
**주제**: Upgrade Plan 오류 처리 UI 분석 및 "쉬운 수정" 구현 아이디어

---

## 1. 현재 화면 분석 (UI/UX 검증)

사용자께서 공유해주신 스크린샷(오류 상태)에 대한 분석 결과입니다.

### ✅ 의도된 설계 (정상 작동)

현재 두 가지 "🔁 재평가" 버튼이 공존하는 것은 버그가 아니며, **상황에 따른 맥락적 유도**를 위한 의도된 설계입니다.

| 구분     | 상단 버튼 (파란색)         | 하단 버튼 (주황색)                |
| :------- | :------------------------- | :-------------------------------- |
| **색상** | `bg-blue-100` (Blue)       | `bg-amber-100` (Amber)            |
| **의미** | **능동적 재평가**          | **위험/오류 복구 (Retry)**        |
| **상황** | 결과 불만족 시 언제든 사용 | 시스템 오류로 결과가 없을 때 사용 |
| **목적** | "새로운 결과를 원해!"      | "오류가 났네? 다시 시도해야지."   |

**결론**: 사용자가 현재 상태(오류)를 명확히 인지하고, 즉각적으로 복구 행동(클릭)을 할 수 있도록 유도하는 **안전한 디자인**입니다.

## 2. "How to Fix" 배치 및 "쉬운 수정" 제안

리더님께서 요청하신 **"쉽게 수정(반영)할 수 있는 방법"**에 대한 기술적/디자인적 제안입니다.

### 💡 [BEST] Idea 1: "원클릭 자동 수정" (Auto-Fix)

오류 상황이 해소된 후(Phase 8-F 배포로 해소됨), 가장 권장하는 방식입니다.

- **개념**: 고민 없이 버튼 하나로 즉시 수정
- **위치**: Upgrade Plan 카드 내 `How to Fix` 영역 우측 상단
- **흐름**:
  1. AI가 완벽한 수정본(`patched_content`) 생성
  2. **"⚡ 자동 수정"** 버튼 표시
  3. 클릭 시 → 에디터의 문제 문장을 **자동 교체**

### 💡 Idea 2: "비교 보기 후 적용" (Diff View)

수정 내용을 더 신중하게 확인하고 싶을 때 적합합니다.

- **개념**: `[수정 전] vs [수정 후]` 비교 팝업 제공
- **위치**: "How to Fix" 내용 클릭 시 팝업 호출
- **흐름**:
  1. 변경 사항을 붉은색(삭제)/초록색(추가)으로 시각화
  2. 사용자가 눈으로 확인 후 **"적용하기"** 버튼 클릭

### 💡 Idea 3: "드래그 앤 드롭" (Drag & Drop)

사용자가 원하는 문장만 골라서 적용하고 싶을 때 유용합니다.

- **개념**: How to Fix의 텍스트를 마우스로 끌어서 에디터로 이동
- **장점**: 직관적이고 물리적인 수정 경험 제공

## 3. 향후 계획 (Next Steps)

현재 **Phase 8-F (참고자료 연결)** 배포가 완료되었으므로, 다음과 같은 변화가 예상됩니다:

1. 하단의 **주황색 버튼(오류)**이 사라지고, 정상적인 **파란색 Upgrade Plan**이 표시됩니다.
2. Upgrade Plan에는 사용자가 업로드한 **참고자료의 스타일**이 반영됩니다.
3. 정상 상태가 되면 숨겨져 있던 **"⚡ 자동 수정" 버튼**이 나타나, **원클릭 수정**이 가능해집니다.

**제안**: 프로덕션 배포가 완료되었으므로, 다시 한번 테스트해보시면 **"오류가 사라지고 수정 버튼이 생기는"** 극적인 변화를 경험하실 수 있습니다.
